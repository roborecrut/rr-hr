
# План оптимизации ЛК работодателя

Большой объём — разбиваю по приоритетам. Подтверди — реализую блоками.

## 1. Ширина и навигация (быстро)
- Увеличить max-width контентной области ЛК на ~20–25% (с текущих ~1280 до ~1560–1600px).
- В CRM «Канбан» сделать горизонтальную прокрутку с **закреплённым внизу экрана** скроллбаром (sticky bottom scrollbar поверх вьюпорта), чтобы не нужно было листать вниз.
- В «Канбан» уменьшить ширину карточек/колонок, чтобы они не были такими растянутыми.

## 2. Кнопка «···» (FieldActionsMenu)
- Вернуть видимую иконку (сейчас визуально пропала — починю SVG/контраст).
- Убрать сдвиг контента при открытии (Radix Portal + правильные стили, без занятия места).

## 3. Профиль работодателя
- Удалить блок «Профиль работодателя» (скрин №5 — дублирующий заголовок).
- Поднять блок «Аккаунт Google» (скрин №6) на первое место, под ним — реферальная ссылка.

## 4. Меню/баланс (типографика)
- Увеличить шрифт пунктов «Пульт Управления Рекрутом» и блока «Баланс и лимиты» (≈ +15–20%).

## 5. Карточка кандидата (главный приоритет)
- Убрать техническую информацию (id, токены, raw json, технические таймстемпы).
- Увеличить шрифты и размеры форм, более воздушный layout.
- **Скрыть пустые секции** по умолчанию (auto-collapse, если значение пустое).
- Текст резюме (распознавание) — большая читабельная область (моноширинный убрать, font-size 15–16px, max-height + scroll).
- Яркий статус-бейдж: «✓ Одобрен ИИ» (зелёный), «Подходит частично» (жёлтый), «Не подходит» (красный) на основе среднего балла.
- Цветовая разметка оценок ИИ:
  - зелёный — хорошо (≥ 70 / 100 либо «good»),
  - жёлтый — средне (40–69 либо 2–3 из 5),
  - красный — плохо (< 40 либо «bad»).
- Применить к каждому комментарию/оценке в чек-листе, ситуациях, резюме.

## 6. Всплывающие подтверждения
- Убрать нативный `window.confirm` («Подтвердите действие на hr-rr.online», скрин №3) — заменить на тихое действие либо in-app тост с Undo. Затронет: «Шаблон должности», «Очистить поле», возможные «Удалить».

## 7. Ошибки ИИ (скрин №4 «Я сломался…»)
- Аудит вызовов AI: `aiClient.invoke` ловит сообщение «Не удалось подтвердить вход» — это auth race при первом вызове edge function.
- Фикс: ретрай 1 раз после `supabase.auth.getSession()` если ошибка `Invalid JWT/no session`.
- Не показывать generic-«сломался» при бизнес-ошибках 402/429 — показывать понятные тексты.

## 8. Редактор интервью (скрин №11)
- Над каждым полем добавить пояснительный заголовок:
  - **Резюме**: «Промт для ИИ — что учитывать при оценке резюме» / «Критерии, по которым ИИ оценит резюме кандидата».
  - **Чек-лист**: «Промт для ИИ — какие навыки проверять» / «Чек-лист, который проходит кандидат, ИИ оценивает каждый пункт».
  - **Ситуации**: «Промт для ИИ — какие сценарии генерировать» / «Ситуации, на которые отвечает кандидат; ИИ оценивает соответствие критериям».
- Визуально разделить «промт» и «результат» (разные иконки/цвет рамки, подпись «Вход для ИИ» vs «Результат ИИ»).

## 9. Тарифы и Счета — разбить на 4 вкладки
В существующем разделе оставить одну запись в меню, внутри — Tabs:
1. **Баланс** — текущий баланс и лимиты.
2. **Покупка и оплата** — пакеты, кнопка оплатить.
3. **История списаний** — таблица транзакций.
4. **Реферальная программа** — ссылка, статистика, выплаты.

## Технические детали
- Контейнер: переменная `--panel-max-w: 1560px` в `index.css`, применяется в `EmployerPanel.tsx` и страницах редакторов.
- Sticky scrollbar для канбана: обёртка с `position: sticky; bottom: 0` + проброс scrollLeft через ResizeObserver, либо CSS-only через overflow-wrapper.
- FieldActionsMenu: проверить, что DropdownMenuTrigger не схлопывается (отсутствие иконки = SVG stroke=currentColor конфликтует с золотым фоном).
- Цветовая разметка ИИ: утилита `scoreTone(value, max)` → `{cls, label}`, применяется в `CandidateDetailsModal.tsx` / `CandidateDocsDossier.tsx`.
- Замена `window.confirm` → in-app `AlertDialog` (shadcn) или сразу действие с тостом «Отменить».
- Auth race в `aiClient.ts`: перед `functions.invoke` дождаться `await supabase.auth.getSession()`; при 401 — один автоматический retry.
