## Подключение Робокассы + оферта + реквизиты

### 1. База данных (миграция)
- В `employers` добавить:
  - `offer_accepted boolean default false`
  - `offer_accepted_at timestamptz`
  - `offer_version text` (например `'2026-06-06'`)
- Таблица `payments_robokassa`:
  - `id`, `employer_id`, `inv_id` (bigserial, уникальный), `amount_rub numeric`, `status` (`pending|paid|failed|refunded`), `signature_in/out`, `raw_payload jsonb`, `paid_at`, `created_at`, `updated_at`.
- RPC `robokassa_create_invoice(_amount_rub int)` — создаёт запись `pending`, возвращает `{ inv_id, amount }` (вызывается из фронта залогиненным работодателем).
- RPC `robokassa_mark_paid(_inv_id bigint)` — security definer, вызывается **только** из edge-функции (через service role); идемпотентно начисляет RR через `apply_transaction(..., 'topup', amount, ...)` и помечает счёт `paid`.
- GRANTs + RLS: работодатель видит только свои счета.

### 2. Edge-функции
- `robokassa-result` (публичная, `verify_jwt=false`) — приёмник ResultURL от Робокассы. Проверяет подпись `MD5(OutSum:InvId:Password2[:Shp_*=...])`, дергает `robokassa_mark_paid`, отвечает `OK<InvId>`.
- `robokassa-check` (опционально) — проверка статуса по `OpState` через серверный API Робокассы для ручной сверки.
- Секреты (запросим позже через add_secret, не сейчас):
  - `ROBOKASSA_LOGIN`, `ROBOKASSA_PASSWORD1`, `ROBOKASSA_PASSWORD2`, `ROBOKASSA_TEST_PASSWORD1`, `ROBOKASSA_TEST_PASSWORD2`, `ROBOKASSA_IS_TEST` (`1`/`0`).

### 3. Фронт — оплата
- `src/lib/robokassa.ts` — функция формирования URL `https://auth.robokassa.ru/Merchant/Index.aspx` с параметрами `MerchantLogin, OutSum, InvId, Description, SignatureValue=MD5(login:OutSum:InvId:pwd1)`, `IsTest`. **Подпись для входящего URL формируем на сервере** в новой edge-функции `robokassa-create` (чтобы не светить pwd1 в браузере) — возвращает готовый `payment_url`.
- В `EmployerPanel.tsx` (блок «Пополнение баланса RR за рубли»):
  - Кнопка «Пополнить баланс» вместо текущего `topup_rr` RPC вызывает `robokassa-create` → редирект на полученный URL.
  - Над кнопкой: чекбокс «Я ознакомлен и согласен с [Публичной офертой](popup)» — по умолчанию включён; без галочки кнопка disabled. Текст оферты открывается в модалке со скроллом до низа.
  - Передаём флаг `offer_accepted=true` при создании счёта (RPC `robokassa_create_invoice` пишет в `employers.offer_accepted` если ещё не стоял).

### 4. Фронт — оферта и согласие при регистрации
- Страница `/offer` — публичная, рендерит размеченный по блокам текст оферты (из PDF) в стиле сайта (тёмный фон, золотые акценты, карточки разделов). Заголовок H1, секции H2, контакты в финальной карточке. Доп. формулировки добавим (территория, контактные данные оператора, согласие на обработку ПДн со ссылкой на политику — короткий абзац для соответствия требованиям Робокассы).
- Компонент `OfferConsentCheckbox` (переиспользуемый):
  - Чекбокс по умолчанию включён.
  - Текст: «Регистрируясь / Оплачивая, я подтверждаю, что ознакомлен(а) с [публичной офертой](…) и принимаю её условия».
  - Ссылка открывает попап `OfferDialog` со скроллом; кнопка «Принять» активна только после скролла до низа.
- В `AuthModal.tsx` встроить чекбокс под Google-кнопкой; передавать `offer_accepted` в метаданные при регистрации (через `signup-bootstrap` или существующий триггер `handle_new_user`). Дополнительно: после первого входа `employers` обновляется флагом `offer_accepted=true, offer_accepted_at=now()`.
