### Пункт 2. Админ-страница `/admin` для управления шаблонами должностей

**Доступ:** только пользователи с ролью `admin` (проверка через `has_role(auth.uid(), 'admin')`). Для остальных — редирект на `/` с тостом «нет доступа».

**Маршрут:** добавить `/admin` в `src/App.tsx`, новая страница `src/pages/AdminPanel.tsx`.

**UI разделы (вкладки):**
1. **Должности и шаблоны** (основной раздел этого пункта):
   - Слева — список всех записей из `job_titles` с поиском по названию, индикатором «сколько полей заполнено» (например, `8/13`), счётчиком использований (`usage_count`), фильтром «базовые / пользовательские / без шаблонов».
   - Справа — редактор выбранной должности: 13 текстовых полей (7 вакансия + 6 обучение), сгруппированных по секциям, каждое с кнопкой «Очистить» и «Сгенерировать через AI» (использует существующий `ai-enhance` Edge Function).
   - Кнопки сверху: «Сохранить», «Удалить должность» (с подтверждением), «Сделать базовой» (toggle `is_basic`), «Добавить новую должность».
   - При сохранении — прямой `UPDATE` через сервис-RPC `admin_job_title_upsert_templates` (см. ниже), который для админа **перезаписывает** поля (в отличие от обычного `job_title_save_templates`, который только дополняет).
2. **Стартовые наборы** (вспомогательное): кнопка «Сгенерировать шаблоны для пустых должностей через AI» — пакетная операция (последовательно по 1–2 шт., чтобы не упереться в rate limit).

**Новые БД-объекты (миграция):**
- RPC `admin_job_title_upsert_templates(_title text, _patch jsonb, _overwrite boolean)` — `SECURITY DEFINER`, проверяет `has_role(auth.uid(), 'admin')`, при `_overwrite = true` полностью перезаписывает указанные ключи, при `false` ведёт себя как обычный `job_title_save_templates`.
- RPC `admin_list_job_titles()` — возвращает все должности с `field_templates`, отсортированные по `usage_count desc`. SECURITY DEFINER + проверка admin.
- RPC `admin_delete_job_title(_id uuid)` — удаление, admin only.

### Технические детали

- Все RPC — `SECURITY DEFINER` с `SET search_path = public`.
- В админ-RPC первым делом: `IF NOT public.has_role(auth.uid(),'admin') THEN RAISE EXCEPTION 'forbidden'; END IF;`.
- В `AdminPanel.tsx` дополнительная клиентская проверка роли через запрос к `user_roles` (UI-гейт), но реальная защита — на уровне RPC.
- AI-генерация одиночного поля в админке — переиспользует существующий `aiClient.enhanceField()` с `template` пустым (поскольку мы и создаём template).
- Пакетная генерация в разделе «Стартовые наборы» — простой `for...of` по списку с задержкой 500 мс между вызовами.
