## Цель

Сделать личный кабинет работодателя легче и читабельнее: крупнее текст и поля, меньше визуального шума, групповые действия — в меню «···». Стиль и бренд (Apple Liquid Glass, синий градиент, золотые акценты) сохраняем — меняем только плотность, типографику и группировку.

## Что меняем

### 1. Общая типографика и плотность (`src/index.css` + `.brand-editor`)

- Базовый размер текста в редакторах: `text-xs` (12px) → **`text-sm` (14px)**, подписи `text-[10px]` → **`text-[11–12px]`**.
- Поля ввода/textarea: высота/паддинги +30%, `text-sm`, `leading-relaxed`.
- Заголовки секций: крупнее, больше воздуха (отступы между секциями `space-y-6` → `space-y-8`).
- Отказаться от `font-mono` в textarea редактора вакансии — обычный шрифт читается легче.

### 2. Редактор вакансии (`VacancyEditor.tsx`) — ПРИОРИТЕТ #1

- Каждое поле: три кнопки **AI / Шаблон / Сброс → одно меню «···»** (Popover) в правом верхнем углу карточки поля. AI-улучшение остаётся быстрой иконкой-палочкой рядом, остальное — в меню.
- Превью «Превью на лендинге» — **сворачиваемое** (по умолчанию свёрнуто), раскрывается по клику. Это убирает половину высоты формы.
- Группы полей (`main / motivation / team / training`) — в **аккордеон**: открыта только активная группа, остальные свёрнуты с прогрессом «заполнено N из M».
- Подсказка-шапка «15 полей…» — компактнее, одна строка.

### 3. Раздел Компания (`EmployerPanel.tsx` — вкладка/блок компании, `CompanySections.tsx`)

- Сгруппировать поля в 3 аккордеона: «Основное», «Описание», «Команда и культура».
- Кнопки действий по полю → меню «···».
- Крупнее инпуты, больше воздуха.

### 4. HH-шаблоны (`HHTemplatesSection.tsx`)

- Три textarea сейчас стоят стопкой и занимают экран. Сделать **табы** (Пост / Приглашение / Авторазбор) — видна одна textarea за раз, крупная и читаемая.
- Кнопки «Копировать / AI-улучшить» — в меню «···» у активного таба.

### 5. TrainingWizard и InterviewWizard

- Шапка шага — крупнее, описание короче (одна строка + «подробнее» по клику).
- Внутри шагов: кнопки вторичных действий → меню «···».
- Inputs/textarea — крупнее (как в редакторе вакансии).
- Прогресс шагов — компактная полоска сверху вместо длинного списка.

### 6. Кнопки списков (карточки вакансий/тренингов/интервью)

- В карточках где сейчас 3+ кнопок («Редактировать», «Дублировать», «Пауза», «Удалить» и т.п.) — оставить главную (Редактировать), остальное в меню «···».

## Технические детали

- Новый компонент `src/components/FieldActionsMenu.tsx` — переиспользуемое меню «···» на базе существующего `DropdownMenu`. Принимает массив `{ icon, label, onClick, danger? }`. Используется в `VacancyEditor`, `CompanySections`, wizards, карточках списков.
- `VacancyEditor.FieldRow`: оставить кнопку AI (иконка `Wand2`) видимой при `aiReady && value.length>=7`, остальное (`Шаблон должности`, `Шаблон`, `Сброс`) — в `FieldActionsMenu`.
- Превью: обернуть `PreviewShell` в `<details>` или контролируемый `useState` с кнопкой «Показать превью / Скрыть превью» (по умолчанию скрыто).
- Группы: использовать существующий `Accordion` из `components/ui/accordion.tsx`. Для каждой группы считать `filled / total` по непустым значениям.
- HH-шаблоны: использовать `Tabs` из `components/ui/tabs.tsx`.
- Глобальные размеры — в `.brand-editor` (`src/index.css`): переопределить базовые отступы/размеры; компоненты `text-xs` заменить на `text-sm` точечно там, где это блокирует читаемость.
