## Цель
Устранить гонку запросов: распознавание документа (`ai-ingest-document`) уходит к ProTalk до того, как завершился `/restart`, открытый при входе в редактор. Два параллельных запроса в один диалог ProTalk → ошибка.

## Идея
Завести **глобальный «AI-ready» оверлей с маскотом** в момент открытия любого редактора/мастера, где есть `DocumentUploader`. Пока `/restart` не вернулся — все обращения к нейронке заблокированы, но визуально показывается дружелюбное окно ожидания с кнопкой «Закрыть» (фоновый `/restart` при этом продолжает работать).

## Что меняем

### 1. `src/lib/aiReady.ts`
- Хранить кроме `pending` ещё и состояние «overlay показывается / скрыт пользователем».
- Добавить:
  - `useAIReady()` — без изменений (true когда `pending === 0`).
  - `useAIRestartPending()` — true пока идёт `/restart`.
  - `dismissAIRestartOverlay()` — пользователь закрыл окно; оверлей больше не показываем, но `pending` не трогаем (фоновый запрос продолжается, `useAIReady` станет true сам).
  - `requestAIRestartOverlay()` — сбрасывает «dismissed» при новом запуске.

### 2. Новый компонент `src/components/AIRestartGate.tsx`
- Рендерит fixed-оверлей в стилистике `AIWaitProvider` (синий glass-градиент, золотая рамка, маскот RR7, печатающая фраза «Готовим ИИ к работе…»).
- Показывается когда `useAIRestartPending() && !dismissed`.
- Кнопка-крестик в правом верхнем углу + кнопка «Закрыть и продолжить в фоне» — вызывают `dismissAIRestartOverlay()`. Сам `/restart` не отменяется.
- Без таймера 120с (просто бесконечная анимация — мы не показываем счётчик, чтобы не пугать).

### 3. Хук `useAIRestartOnMount(employerPublicId)`
- Один вызов: при mount компонента редактора/мастера дергает `aiRestart(...)` (это уже инкрементит `pending` через `beginAIRestart`) и `requestAIRestartOverlay()`.
- Возвращает `aiReady` для гейтинга кнопок.

### 4. `DocumentUploader.tsx`
- В `recognize()` перед `aiWaitRun(...)` ждать `aiReady`:
  - если `pending > 0` — показать toast «Подождите, ИИ ещё готовится…» и поллить (или `await` Promise, которое резолвится через listener `aiReady.ts`).
  - кнопка «Распознать документ» получает `disabled={!aiReady}` + подсказка «ИИ готовится…».
- То же для кнопки «Оформить красиво…» (там уже частично есть через `canEnhance`).

### 5. Места, где открываются редакторы/мастера
Уже сейчас в этих местах вызывается `aiRestart(...)`:
- `EmployerPanel.tsx:676` (открытие мастера новой вакансии)
- `EmployerPanel.tsx:745` (мастер компании)
- `EmployerPanel.tsx:1529` (редактор вакансии)
- `VacancyEditor.tsx` / `JobVacancyLanding.tsx`

Действие: рядом с этими `aiRestart(...).catch(...)` вызвать `requestAIRestartOverlay()` чтобы оверлей показался. Оверлей сам исчезнет когда `pending === 0`.

### 6. Монтаж оверлея
- Подключить `<AIRestartGate />` глобально в `src/App.tsx` рядом с `AIWaitProvider`, чтобы работало во всех роутах.

## Поведение для пользователя
1. Открыл «Создать вакансию» → мгновенно появляется маскот «Готовим ИИ к работе…».
2. Если устал ждать — жмёт «Закрыть и продолжить в фоне», окно исчезает, `/restart` продолжается фоном.
3. Жмёт «Распознать документ» → если `/restart` ещё не вернулся, кнопка `disabled` + подсказка. Как только ответ пришёл — кнопка активируется, можно нажимать.
4. Никаких параллельных запросов в ProTalk — гонка устранена.
